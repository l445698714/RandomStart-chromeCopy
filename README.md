## 🌟 核心特性

### 🎯 **智能随机启动**
> 在指定范围内随机选择分身，避免重复启动，支持批量管理

- **🎲 智能随机算法** - 在1-100或自定义范围内随机选择分身启动
- **📊 三行实时状态** - 显示正在启动、已启动、未启动分身的清晰状态
- **🔄 避重复启动** - 自动记录已启动分身，避免重复操作
- **⚡ 可调延迟控制** - 自定义启动间隔，防止系统过载

### 🎨 **专业图标替换**
> 为每个Chrome分身设置独特图标，便于快速识别和管理（感谢@Devilflasher)

- **🖼️ 精美预设图标** - 专业设计的Chrome分身图标
- **♾️ 无限动态生成** - 超过45的编号自动生成带数字的圆形图标
- **🔄 一键自动应用** - 启动分身后自动应用对应编号图标
- **🎯 手动图标管理** - 支持为指定范围分身批量更新图标
- **📂 快捷方式更新** - 同时更新桌面快捷方式图标，保持一致性

### 🚀 **批量分身创建**
> 快速创建大量Chrome分身配置文件和快捷方式

- **📁 自动配置管理** - 自动创建独立的Chrome用户数据目录
- **🔗 快捷方式生成** - 批量生成数字命名的分身快捷方式
- **⚙️ 参数自定义** - 支持自定义Chrome启动参数和端口配置
- **🎯 路径智能管理** - 自动管理缓存路径和快捷方式目录

## 📖 详细使用指南

### 🎯 **随机启动功能**

1. **设置启动范围** - 在"启动范围设置"中输入起始和结束编号（如1-100）
2. **设置启动数量** - 在"启动数量"中输入要启动的分身数量（如5）
3. **点击随机启动** - 程序将随机选择5个分身启动
4. **查看状态** - 主界面显示三行状态：
   ```
   🎯 正在启动分身：23
   ✅ 已启动分身：12、34、56、78、90
   ❌ 未启动分身：1、2、3、4、5...
   ```

### 🎨 **图标替换功能**

#### 自动图标应用
- **启用自动应用** - 勾选"启动后自动应用图标"选项
- **启动分身** - 使用任意启动方式启动分身
- **自动替换** - 程序自动为每个分身应用对应编号的图标

#### 手动图标管理  
1. **进入图标管理** - 点击"图标管理"标签页
2. **设置操作范围** - 输入要更新图标的分身编号范围
3. **选择操作类型**：
   - **生成图标** - 为指定范围生成新图标文件
   - **应用图标** - 为当前运行的分身应用图标
   - **更新快捷方式** - 更新桌面快捷方式图标
4. **执行操作** - 点击对应按钮执行操作

### 🚀 **批量创建分身**

1. **设置缓存路径** - 指定Chrome用户数据存储位置
2. **设置快捷方式路径** - 指定快捷方式文件保存位置  
3. **输入创建范围** - 在"指定范围"中输入要创建的分身编号（如1-50）
4. **开始创建** - 点击"指定启动"会自动创建不存在的分身配置

## ⚙️ 高级配置

### **启动参数自定义**
程序自动为每个分身配置独立的：
- 用户数据目录 (`--user-data-dir`)
- 调试端口 (`--remote-debugging-port`)
- 进程隔离参数

### **图标资源管理**
- **图标位置**: `icons/chrome_1.ico` ~ `icons/chrome_45.ico` (预设计图标)
- **动态生成**: 编号>45时自动生成圆形数字图标 (chrome_46.ico, chrome_100.ico...)
- **支持格式**: ICO格式，多尺寸图标
- **自动缓存**: 生成的图标自动缓存，避免重复生成

### **状态持久化**
程序自动保存：
- 已启动分身列表
- 用户界面设置
- 路径配置
- 启动参数设置

## ❓ 常见问题

### **Q: 分身启动失败？**
**A:** 检查快捷方式路径设置，确保存在对应编号的.lnk文件

### **Q: 图标不显示？**
**A:** 确保程序以管理员权限运行，检查icons目录是否完整

### **Q: 如何重置所有设置？**
**A:** 删除程序同目录下的配置文件，重新启动程序

### **Q: 支持多少个分身？**
**A:** 理论上支持无限数量分身，建议不超过100个以保证性能

## ⚠️ 注意事项

- **文件命名**: 快捷方式必须以数字命名（1.lnk, 2.lnk...）
- **管理员权限**: 图标替换功能需要管理员权限
- **系统资源**: 同时启动大量分身会占用较多系统资源
- **Chrome版本**: 建议使用最新版本Chrome浏览器
